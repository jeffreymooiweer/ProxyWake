# ProxyWake v4.2.1

## Summary

Patch release focused on repository polish, version consistency, documentation cleanup, and release reliability. No new product features and no breaking API changes.

## Added

- `scripts/check_version_consistency.py` — automated version alignment verification
- `.github/workflows/version-check.yml` — CI gate for version drift
- `scripts/version_utils.py` — shared version helpers for sync and CI
- Expanded `docs/` documentation pages and examples
- `docs/github-metadata-checklist.md` for GitHub repository settings
- Root `.env.example` for Docker Compose
- Tests for groups, public routes, API write scopes, and config helpers

## Changed

- Repository version references aligned to **4.2.1**
- `backend/version.py` exposes `__version__` and `VERSION`
- README and docs install examples use `jeffersonmouze/proxywake:4.2.1`
- `SECURITY.md` supported versions policy clarified
- Docker workflow derives default-branch tags from `backend/version.py`
- `PROXYWAKE_SESSION_COOKIE_SECURE` for HTTPS-only deployments

## Fixed

- CHANGELOG compare links (`[Unreleased]`, `[4.2.1]`)
- Stale v4.2.0 treated as current release in documentation
- Hardcoded Docker workflow tags replaced with version-derived tags
- Changelog claims verified against actual repository files

## Upgrade Notes

- No database migration required.
- No breaking API changes.
- Existing Docker users can pull `jeffersonmouze/proxywake:4.2.1` and recreate the container with the same volume.

```bash
docker pull jeffersonmouze/proxywake:4.2.1
docker stop proxywake && docker rm proxywake
# Re-run docker run or docker compose with the same env and volume
```
