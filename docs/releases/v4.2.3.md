# ProxyWake v4.2.3

## Summary

Patch release fixing Unraid and similar deployments where the appdata volume is not writable by the container user.

## Fixed

- Docker entrypoint runs as root on startup, fixes ownership of `PROXYWAKE_DATA_DIR`, then starts gunicorn as `proxywake` (UID 1000)
- Resolves `sqlite3.OperationalError: unable to open database file` on first Unraid install

## Upgrade

```bash
docker pull jeffersonmouze/proxywake:4.2.3
```

Recreate the container with the same appdata path and environment variables.

**Workaround without upgrading:** `chown -R 1000:1000 /mnt/user/appdata/proxywake`
